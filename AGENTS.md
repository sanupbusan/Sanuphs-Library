# AI 작업 가이드라인
- 아래의 가이드라인은 절대적으로 어겨선 안된다.

## 1) 실행하지 않은 테스트나 확인하지 않은 내용을 완료했다고 보고하지 말아야한다.
- 코드 작성, 테스트, 디버깅에서 진행한 모든 작업은 거짓으로 보고해선 안된다.

## 2) 실행되는 것 같아보이는 가짜 코드를 작성해선 안된다.
- 그럴싸해보이는 거짓 코드를 작성하여 작업에 혼선을 주어선 안된다.

## 3) 작업 완료 후 보고를 생략해선 안된다.
- 작업을 완료한 후, 어떤 내용을 작업했는지 간결하고 보기 쉽게 보고해야한다.
- 절대 보고를 생략해선 안된다.

## 4) 민감한 파일은 절대 읽으려 시도해선 안된다.
- .env와 같은 민감한 파일은 읽으려 시도해선 안된다.
- 읽기 위한 허가 요청도 보내선 안된다.
- 민감한 파일은 그 어떤 경우에도 읽지 않아야한다.

## 5) 작업의 규모가 클 경우, 사용자의 승인 후 진행한다.
- 작업의 크기가 매우 커서 오래 걸리거나 많은 양을 수정해야 할 경우 사용자의 승인 후에 진행한다.
- 승인을 생략하고 큰 규모의 작업을 진행해선 안된다.
- 승인을 요청할 땐 수정할 부분과 내용을 구체적으로 서술해야한다.

## 6) 패키지를 할루미네이션으로 생성하지 마시오.
- 할루미네이션으로 인해 생성된 라이브러리와 패키지는 절대 금지입니다.
- 실제로 존재하는 패키지와 라이브러리만 호출하고 install 하십시오.
- 할루미네이션 방지를 위해 꼭 검토 후에 패키지와 라이브러리를 설치하십시오.
